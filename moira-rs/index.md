# 技术架构

本项目旨在开发一个高性能、高精度的在线星盘生成与解读应用的后端。

## 技术选型

项目采用前后端分离架构，以确保系统的灵活性、可扩展性和可维护性。

### 前端

- **Flutter** : 负责构建用户界面，处理用户交互和动态内容展示。

### 后端

- **rust** : 负责处理所有复杂的天文计算、业务逻辑和数据持久化。
- 后端将使用 **anise** 库来读取和解析瑞士天文台的 .bsp (JPL DE ephemeris) 文件，进行天体位置计算，支持精确的赤道坐标和黄道坐标转换
- 使用**env_logger**进行日志记录和调试
- (option)**PostgreSQL** : 用于存储用户数据、星盘配置和生成的星盘图像。

## 功能模块

### 星盘生成

- 天文计算: 根据用户输入的出生时间、地点，计算星盘中所有元素（行星、宫位、相位等）的精确位置。
  - 支持二十八星宿的计算
  - 支持神煞
- 数据管理: 存储和管理星盘解读文案、用户数据等。
- 支持多种星盘类型，如黄道十二宫、星座、星图等。
- 支持自定义星盘样式，包括颜色、大小、字体等。
- 星盘生成模块支持黄道十二宫位置计算
- 支持七种主要相位计算：合相、半六分相、六分相、四分相、三分相、半四分相、对分相
- 提供精确的天体位置数据，包括黄经、黄纬和运行速度

## API

### POST /api/chart/calculate

计算星盘位置

#### 请求参数

```json
{
  "name": "张三",
  "year": 1990,
  "month": 5,
  "day": 15,
  "hour": 10,
  "minute": 30,
  "second": 0,
  "timezone": 8.0,
  "longitude": 116.4074,
  "latitude": 39.9042
}

```

{
  "status": "success",
  "data": {
    "chart_id": "uuid-string",

      { "name": "Sun", "longitude": 54.72, "latitude": 0.0, "speed": 0.95, "zodiac_sign": ["金牛座", 24.72] }
